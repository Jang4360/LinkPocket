# learning — 학습 기록

이 폴더에는 두 종류만 있다.

| 파일/폴더 | 무엇 |
|---|---|
| [cs-learning.md](cs-learning.md) | **학습 로드맵.** P0/P1 8축 매트릭스(A~H)와 주차별 실행표. "무엇을 언제 배우고 무엇을 증거로 남기는가"의 지도. |
| [articles/](articles/) | **학습 아티클.** 정해진 범위의 주제 하나를 면접에서 그대로 말할 수 있게 깊게 정리한 글. (일일 단상용 `til`은 아티클과 겹쳐 제거함 — 짧은 메모는 작성 중 아티클의 초안 섹션으로 쌓는다.) |

## 학습 글을 쓰기 전에 반드시 읽는 것

1. **[cs-learning.md](cs-learning.md)** — 쓰려는 주제가 어느 축(A~H)에 속하는지, 그 축의 "학습할 것 · 적용할 것 · **검증·기록할 증거**"가 무엇인지 확인한다. 아티클은 이 세 칸을 풀어 쓴 것이다.
2. **[articles/README.md](articles/README.md)** — 이 폴더 글의 **말투·품질 규칙과 템플릿**. 구조와 톤을 여기에 맞춘다.

## 원칙

- 주제는 **질문형이 아니라 범위가 정해진 것**이어야 한다. 예: `명세·계약과 TDD 기반 테스트 자동화`, `SSRF와 외부 URL 수집 보안` (cs-learning의 A~H 축 또는 그 소항목).
- 모든 학습 글은 **정해진 범위 → LinkPocket 적용 → 남길 증거**로 끝난다. 일반 블로그 글과 이 폴더 글을 가르는 선은 "LinkPocket의 실제 문제와 증거에 붙어 있는가"이다.
- 발행한 글의 외부 링크는 [reference/](../reference/)에 모아 회사 유형별 서사와 매핑한다.
